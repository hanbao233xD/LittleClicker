# 安卓自动点击器开发计划书（AI 协作版）

最后更新：2026-03-22
适用项目：LittleClicker（Kotlin + Jetpack Compose）

## 目标
将“安卓自动点击器”按阶段推进为可交付产品，并沉淀一套可反复使用的 AI 协作流程。

## 协作原则
1. 严格阶段化推进：上一阶段可运行并验证后，再进入下一阶段。
2. 报错即回传：出现错误时，将完整日志提供给 AI 定位修复。
3. 单阶段迭代：实际开发过程中，按“你提出一个阶段，AI实现一个阶段”执行，不跨阶段一次性堆功能。
4. 文档同步：每个关键改动完成后，同步更新 `docs/`。

## 执行流程（每次开发）
1. 你给出当前要做的单一功能。
2. AI 只实现该功能及其必要依赖，不抢跑后续内容。
3. 本地编译/运行验证。
4. 若失败：贴完整报错日志给 AI。
5. 若通过：记录到 `docs/IMPLEMENTATION_LOG.md`，再进入下一个功能。

## 详细阶段提示词（原版归档）

### 阶段一：项目初始化与底部导航架构
我正在开发一个面向小白用户的安卓自动点击器 App，开发语言是 Kotlin，UI 框架使用 Jetpack Compose。
请提供 build.gradle.kts 配置。引入 Compose 基础依赖、androidx.navigation:navigation-compose，以及 top.yukonga.miuix.kmp:miuix-android:0.8.7 这个第三方 UI 库，还有处理 JSON 序列化的 Gson 库。
提供 AndroidManifest.xml 的完整配置，加上悬浮窗、无障碍服务和忽略电池优化的权限声明。
写一个 MainActivity.kt。使用 Compose 的 Scaffold 和 NavHost 搭建一个带底部导航栏的架构。底部导航必须包含三个 Tab：【首页】、【脚本管理】、【关于】。
【首页】UI 要求：采用 miuix 风格，提供权限“保姆级”引导。用卡片展示悬浮窗、无障碍、电池优化的授权状态，点击跳转系统设置。全部授权后，底部亮起“开启悬浮窗”按钮。
【关于】UI 要求：居中展示 App 图标、名称、版本号，下方排列“检查更新”、“隐私政策”、“免责声明”等列表项，整体视觉要简约高级。

### 阶段二：可视化悬浮窗与打靶系统
第一阶段跑通了。现在进行【阶段二】，请帮我实现对用户极其友好的悬浮窗交互。
创建 FloatingWindowService.kt (继承 LifecycleService)。
使用 WindowManager 配合 ComposeView 添加悬浮控制面板。
控制面板 UI（miuix 风格）：半透明深色背景，大圆角。包含四个功能键：🔴录制（醒目）、▶️播放/暂停、⚙️设置、❌关闭。
靶标系统：写一个添加“点击靶标”的逻辑。用户可以添加多个靶标在屏幕上拖动。靶标的 UI 是一个带准星的圆圈，并且圆圈中心必须带有递增的数字序号（1, 2, 3...），让小白清楚知道点击顺序。
提供相关的 Kotlin Compose 代码。

### 阶段三：核心引擎与安全机制
悬浮窗和靶标很直观了。现在进行【阶段三】，完善底层的点击与滑动引擎。
创建 AutoClickService.kt 继承 AccessibilityService。
写一个 fun click(x: Int, y: Int) 方法，使用 GestureDescription API 实现点击。
写一个 fun swipe(startX: Int, startY: Int, endX: Int, endY: Int, duration: Long) 方法实现滑动。
核心安全体验：在 Service 中重写 onKeyEvent，当检测到按下“音量减小”键时，立刻清空并终止所有正在执行的点击任务，并在屏幕下方弹出一个 Toast 提示“已紧急停止”。

### 阶段四：傻瓜式录制与文件化保存
引擎配置完毕。现在进行【阶段四】，要求把录制结果保存为外部可见的本地 JSON 文件。
在 FloatingWindowService 中实现录制逻辑：点击面板“🔴录制”后，盖一层全屏透明的 View 拦截触摸事件（MotionEvent），提取坐标和时间间隔。
定义数据模型：用 Kotlin data class 写 Action（包含坐标、延迟时间等）和 Script（包含脚本名、动作列表）类。
录制结束交互：录制完毕弹出一个 Dialog 提示用户输入【脚本名称】。
文件保存逻辑：写一个 ScriptFileManager.kt 工具类。
使用 Gson 将 Script 对象序列化为 JSON 字符串。
将 JSON 文件保存到手机外部存储的 Documents 目录下，确保用户可以用系统自带的文件管理器直接看到并手动编辑。

### 阶段五：脚本管理页与精准定时
录制和文件保存都搞定了。现在进行【阶段五】，请帮我完善底部导航中【脚本管理】Tab 页面的内容。
列表渲染：调用 ScriptFileManager 读取 Documents 目录下的所有 .json 脚本文件，用 Compose 的 LazyColumn 展示。每个列表项要显示脚本名、动作数量。
脚本编辑功能：点击列表项的“编辑”图标进入子页面。用列表展示该脚本的所有动作（Action）。用户可以手动修改特定步骤的 X、Y 坐标和延迟时间。保存后覆盖原 .json 文件。
任务配置面板：点击列表项主体时，弹出底部面板 (Bottom Sheet)，提供“循环次数”和“点击间隔”配置，并有一个“加载到悬浮窗”的按钮。
结合自定义 NTP 时间同步：在【关于】页加个输入框配置 NTP 服务器（默认 ntp.aliyun.com）。写一段基于协程 UDP 请求该服务器的 Kotlin 代码。若用户设置了定时，悬浮窗显示倒计时，时间一到立刻执行脚本。

### 阶段六：防检测机制与内存优化
最后的收尾阶段【阶段六】，做防检测优化和内存收尾。
让点击看起来像真人：修改核心点击方法，利用 Random.nextInt() 让 X 和 Y 坐标随机偏移 -4 到 +4 个像素。
每次动作的延迟时间也随机加入 -15 到 +15 毫秒的波动。
防闪退处理：在悬浮窗的 Compose UI 外部加一层 try-catch 保护。
内存防泄漏：检查 FloatingWindowService 和 AutoClickService，提供在 onDestroy 中调用 composeView.disposeComposition() 并彻底 removeView 的代码，确保用户长时间挂机不会导致内存溢出。

### 阶段七：高阶生态功能
基础功能都很完美了。现在进行最后的【阶段七】，增加脚本社交分享和高级交互。
一键分享与导入：在【脚本管理】列表页给每个脚本增加“分享”按钮，调用 Android 的 Intent.ACTION_SEND 将 JSON 文件分享。增加“导入”按钮，调用系统文件选择器读取别人发来的 JSON。
长按手势支持：扩展 Action 数据类，增加 LONG_PRESS（长按）类型。在 AutoClickService 中使用 GestureDescription 实现长按逻辑。
关联 App 自动唤醒：在脚本配置中增加“关联应用”选项。在无障碍服务里监听前台应用包名变化，当用户打开关联的 App 时，自动弹出悬浮窗并准备好对应的脚本。
