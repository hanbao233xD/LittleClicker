# LittleClicker AI 记忆文档

最后更新：2026-03-22

## 用户长期要求（仅记录要求）
1. 在项目中维护 `docs/` 目录，用于记录 AI 生成和修改的关键内容。
2. 每次完成较重要的实现、结构调整或需求变更时，同步更新 `docs` 下对应文档。
3. 新需求若与已有实现相关，优先先阅读并更新 `docs`，保证上下文连续可追溯。
4. `AI_MEMORY.md` 只负责记录用户要求，不记录实现细节、模块说明、改动清单。
5. 实现细节、模块作用、变更记录需写入其他文档（如 `MODULES.md`、`IMPLEMENTATION_LOG.md`）。
6. 你提供的“安卓自动点击器七阶段提示词”已固化为长期计划书，后续可复用。
7. 实际开发节奏固定为：你提一个阶段，AI写一个阶段；不跨阶段抢跑。
8. 以后 AI 写代码的经验都要总结到 `docs/` 中，并在后续开发持续执行。
9. 即使开启新对话，也要遵守本项目 `docs/AI_MEMORY.md` 中的要求；开始实现前先读取并对齐这些要求。
